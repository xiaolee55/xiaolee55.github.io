每天写代码都会用到不同类型的变量，但是没有细想过这些变量的本质区别，前段时间看了红书有些感悟，所以写篇文纪念一下

先看看两种数据类型以及细分

# 1. 基本类型传值

看下面的代码：

var num1=5;
var num2=num1;

这里先将num1的值置为5，再使用num1来初始化num2，num1和num2之间是完全独立互不影响的，比如在上面代码中加一句并输出

num1=8;
console.log(num2) //输出5

可以发现num1的值变化对num2的值没有影响，他们的内存分析图如下（基本类型的值都是存在栈内存中的）：

(num1的值发生变化前的内存图)

(num1的值发生变化后的内存图)

这是因为num2=num1这一步发生的是num1将其值（本例值为5）的一个副本传递给num2，num1的本体怎么变化都不会影响到那个副本，这和函数的基本类型形参是一个道理。

# 2.引用类型传值

看看下面的代码：

var obj1=new Object();
var obj2=obj1;

这里创建obj2也会在栈中重新开辟一个内存空间，此时obj1指向一个对象（假设为A），obj1的值实际上是一个指针类型（又称引用），所以obj2=obj1实际上是给obj2赋予一个指向对象A的引用，即被引用变量初始化的变量，类型也是引用变量，并且其值是被用于初始化它的引用所指向的对象，可以看一下此时的内存图

我们来简单修改一下，如下：

obj1.name="小明";
console.log(obj2.name);    //输出"小明"

可以看到，代码中修改的是obj1的值，但是obj2的值也随之修改了，这是因为两个变量引用的是同一对象，所有的改变都是共享的，这就好比两人吃同一蛋糕，A咬了一口再给B，B拿到的就是被咬过的蛋糕。

# 传递参数

先说明一点，在JavaScript中，所有函数的参数都是按值传递的，没有按引用传递，即将函数外部的值作为参数传进函数时，在函数内用以接受的参数和传进来的参数并不会共享同一个值，先来看一个基本类型的传参，参看下面的例子：

function add(num){
    num+=10;
    alert(num);    //输出30
}

var count=20;
add(count);
alert(count);    //输出20

显然可以看出，num值的改变对count没有任何影响，因为他们本身就是两个互不干扰的独立变量，num只是一个局部变量，他的生命周期就是函数的执行期间，执行期间num的值会发生变化，但从头到尾都不会对count造成任何影响。

再来看看对于引用类型数据的传参，看下面的例子：

function setName(obj){
    obj.name="小明";
}

var person=new Object();
person.name="小红";
setName(person);
alert(person.name);   //"小明"

很多人在这里会觉得当引用类型数据当做参数时就是按引用传递，他们会觉得函数中obj的改变影响了函数外的person的值，所以会产生误解，实际上，这依然是按值传递，因为所谓的按引用传递，意思是函数中用以接收实参的局部变量的内存空间与实参的内存空间是同一个，但显然不是，这里的obj和person会互相影响完全是因为他们所指向的值是同一个对象，而不是因为他们的内存空间相同。看以下例子：

function setName(obj){
    obj.name="小明";
    obj=new Object();
    obj.name="小李";
}

var person =new Object();
setName(person);
alert(person.name);  //"小明"

这里假设引用类型参数是按引用传递，那么obj参数重新指向一个新对象时person参数也会指向那个参数，因为他们是同一个参数，但实际结果告诉我们，不是这样的，可以看下内存分析：
