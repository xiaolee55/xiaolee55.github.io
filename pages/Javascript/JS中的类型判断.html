# JS中的类型判断

# typeof操作符

# typeof能判断的类型

  • Undefined
  • Object
  • Number
  • String
  • Function
  • Symbol
  • Boolean
  • BigInt
typeof undefined  //undefined
typeof 123        // number
typeof '123'      //string
typeof console.log //function
typeof {}         //object
typeof true       //boolean
typeof 10n        //bigint  (BigInt类型数据不能装箱，因为BigInt只是一个方法，并不是构造函数)

# typeof不能判断的类型

  • Array
  • Null (原因是因为null在底层存储中以全0表示，而000开头表示object，所以错误判断为object)
typeof null  //object
typeof []    //object

# instanceof操作符

  • instanceof操作符主要用来检查构造函数的原型是否在对象的原型链上。包括判断自定义的对象
  • instanceof只要变量的原型链上存在的类型，该变量使用instanceof返回的值都返回true
  • instanceof不能判断基本类型，除非将基本类型装箱（即将基本变量变为对象）
  const s = new String('123');       //字符串封箱并判断
  s instanceof String; // true
  s instanceof Object; // true
  
  s.__proto__ = Object.prototype;   //修改原型后则不属于String类
  s instanceof String; // false
  s instanceof Object; // true
  
    function Animal (name) {         //判断自定义对象
    this.name = name
  }
  const fizz = new Animal('fizz');
  fizz instanceof Animal // true
  
  [] instanceOf Array  //true   判断数组

# constructor属性

''.constructor   //String
(1).constructor  //Number
true.constructor     //Boolean
new Function().constructor  //Function
new Date().constructor  //Date

# 缺点

  • null和Undefined是无效的对象，所以没有constructor存在，所以通过这个无法判断
  • constructor属性可以被随意修改，而且在原型继承中，很容易忽略掉constructor的正确指向

# toString方法（终极解决方案）

function type (obj) { 
    return Object.prototype.toString.call(obj).slice(8, -1).toLowerCase()
}
console.log(type(null))     
console.log(type(1)) 
console.log(type('zenquan')) 
console.log(type(undefined)) 
console.log(type(Symbol())) 
console.log(type(true)) 
console.log(type(console.log))
  • 缺点：不能判断自定义类型，解决方案：使用Symbol.toStringTag
class User{ 
//...
     get [Symbol.toStringTag](){ 
         return 'User'; 
    }
}
 class VipUser extends User{ 
     constructor(){ 
         super(); 
     } //... 
     get [Symbol.toStringTag](){ 
         return 'VipUser'; 
     } 
 }
let user = new User(), vip = new VipUser()
console.log( Object.prototype.toString.call(user) ); // [object User]
console.log( Object.prototype.toString.call(vip) ); // [object VipUser]

因为使用了 getter 的方式定义，Symbol.toStringTag 属性无法被覆盖
vip[Symbol.toStringTag] = 'adfasdff';
console.log( Object.prototype.toString.call(vip) );		// [object VipUser]
